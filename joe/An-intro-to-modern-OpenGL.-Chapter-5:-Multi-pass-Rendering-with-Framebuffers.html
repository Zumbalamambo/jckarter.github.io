OpenGL extensions

Offscreen rendering wasn't made a standard part of desktop OpenGL until OpenGL 3.0; however, almost every OpenGL 2 implementation supports it as an extension. (In OpenGL ES 2, framebuffer objects are a standard feature.) OpenGL provides a naming convention and capability query system to allow vendors to provide additional features. Extensions are given names of the form GL_vendor_name, where vendor is an all-caps identifier for the vendor, such as NV or ATI. Multi-vendor extensions are given the vendor identifier EXT. OpenGL functions, types, and symbols defined by the extension have the vendor identifier suffixed to their names. The framebuffer extension is named GL_EXT_framebuffer_object, and the functions and symbols it provides have EXT suffixed to their names: glBindFramebufferEXT, GL_FRAMEBUFFER_EXT, and so on.

GLEW makes it easy to check the availability of OpenGL extensions. For most common OpenGL extensions, glewInit will set a global boolean variable GLEW_vendor_name to indicate whether that extension is available under the current OpenGL driver. We can require OpenGL 2.0 with the framebuffer extension using the following code:

    glewInit();
    if (!GLEW_VERSION_2_0) {
        fprintf(stderr, "OpenGL 2.0 not available\n");
        return 1;
    }
    if (!GLEW_EXT_framebuffer_object) {
        fprintf(stderr, "OpenGL framebuffer object extension not available\n");
        return 1;
    }

Framebuffer and renderbuffer objects

The framebuffer object extension introduces two new kinds of OpenGL objects. Framebuffer objects represent an offscreen rendering target. A framebuffer object has attachments that connect it to storage for its color, depth, and stencil buffers. These attachments can be to either texture objects or to renderbuffer objects. Attaching textures to a framebuffer object lets us take the output from one rendering pass and use it as input to another. Renderbuffers provide storage for color, depth, or stencil rendering independent of textures.
